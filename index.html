Cyclistic Bike-Share Analysis

Overview

In this project, I analyzed Cyclistic's bike-share trip data to understand the usage patterns of annual members versus casual riders. The goal was to identify trends and provide strategic recommendations to convert casual riders into annual members, enhancing Cyclistic's profitability and growth.

Business Task

Primary Question: How do annual members and casual riders use Cyclistic bikes differently?

Data Description

  • Data Source: Cyclistic's historical trip data (past 12 months)
  • Key Variables:
    • trip_id: Unique identifier for each trip
    • started_at: Start time of the trip
    • ended_at: End time of the trip
    • member_casual: Membership type (member or casual)
    • ride_length: Duration of the ride in seconds
    • day_of_week: Day of the week the trip started

Methodology

  1. Data Cleaning: Consolidated monthly data tables into a single trips_all table using SQLite.
  2. Data Analysis: Performed descriptive analysis using SQL queries to identify usage patterns.
  3. Visualization: Created insightful visualizations in Tableau to illustrate key findings.

Key Findings

  • Average Ride Length: Members have a longer average ride duration compared to casual riders.
  • Ride Patterns by Day: Members predominantly use bikes on weekdays, aligning with commuting behavior, while casual riders prefer weekends for leisure rides.
  • Total Rides Distribution: Although casual riders account for a higher number of total rides, members contribute more substantial usage per user.
