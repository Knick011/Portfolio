BrainBites: Gamified Learning & Screen Time Management App (Full-Stack Mobile Development)

Existing solutions fall into two camps: screen time tracking apps (passive monitoring without engagement) and educational apps (learning without addressing underlying addiction). No solution combined both to create a virtuous cycle where reducing screen time earns quiz opportunities, and completing quizzes earns time back—a gamified approach that makes productivity rewarding.

Technical Architecture & Implementation

1. Frontend: React Native + TypeScript

The application's user interface was built using React Native 0.80.2 with TypeScript for type safety and improved developer experience. Key architectural decisions include:

  • Component Architecture: Designed modular, reusable component library including HomeScreen, QuizScreen, DailyGoalsScreen, SettingsScreen, LeaderboardScreen, and specialized components like EnhancedMascotDisplay (animated character with 6 emotional states), TimerWidget, ScoreInsightsCard
  • State Management: Implemented Zustand for global state management (score, streaks, quiz state) with custom hooks (useHomeIntegration, useLiveScore) for clean component integration. Used React Context API for theme and localization, ensuring performant re-renders
  • Navigation: Utilized React Navigation with stack-based navigation, deep linking support, and proper TypeScript types for route parameters
  • Animations: Created fluid UI animations using React Native's Animated API for score updates, mascot state transitions, and micro-interactions that enhance user engagement
  • Internationalization: Integrated i18next with react-i18next for comprehensive multi-language support (English, Turkish) with locale switching and proper RTL layout handling

2. Native Android Development: Kotlin Modules

The most technically challenging aspect was implementing device-wide screen time tracking on Android—functionality requiring deep native platform integration that cannot be achieved through JavaScript alone:

  • ScreenTimeService (Foreground Service): Built native Kotlin service running as persistent foreground service with notification, tracking actual device usage across ALL apps using UsageStatsManager API. Implemented second-by-second countdown timer, overtime tracking (penalty system), and real-time event emission to React Native layer via NativeEventEmitter
  • DailyScoreCarryoverService: Engineered sophisticated daily rollover system that calculates bonus points for remaining time (67 points/minute for first 40 minutes, diminishing returns after) and penalty points for overtime (50 points/minute). Runs at midnight via AlarmManager, handles day transitions, persists scores to AsyncStorage with comprehensive logging for debugging
  • BackgroundCheckInReceiver: Implemented AlarmManager-based periodic check-in system firing every 5 hours to maintain engagement, send notifications, and prevent timer tampering
  • NotificationService: Created rich notification system with real-time timer display, action buttons (pause/resume), and custom layouts showing daily score, remaining time, and overtime warnings
  • React Native Bridge: Developed custom Native Module (ScreenTimeModule) exposing Kotlin functionality to JavaScript with methods for starting/stopping/resetting timer, querying current state, and handling permissions (UsageStats access, notification permissions, exact alarms for Android 12+)

3. Service Layer Architecture

Designed comprehensive service-oriented architecture separating business logic from UI components:

  • EnhancedScoreService: Centralized score management handling daily scores, lifetime scores, streak tracking, and persistence to AsyncStorage with transactional updates to prevent data loss
  • HybridTimerService: Abstraction layer coordinating between native timer services and React Native UI, managing timer state, handling day transitions, and ensuring consistency across app restarts
  • TimerIntegrationService: Integration point between quiz system and timer, handling time rewards when quizzes completed, penalty reductions, and maintaining sync between different game mechanics
  • DailyGoalsService: Manages daily goal system (complete 3 hard quizzes, maintain 50% accuracy, check in 3 times, etc.) with progress tracking, time bonus rewards (30-60 seconds per goal), and reset logic
  • NotificationService: Handles all notification scheduling, morning reminders (8 AM daily), check-in notifications (every 5 hours), and congratulatory messages for achievements
  • QuestionService: Quiz question management with categorization (Math, Science, History, etc.), difficulty levels (Easy/Medium/Hard), Turkish and English question banks, and random selection algorithms

  • Streak System: Consecutive correct answers build streaks (up to 10x multiplier), creating dramatic score acceleration and encouraging focused quiz sessions. Highest daily streak tracked for competitive element

Complex Technical Challenges Solved

Challenge 1: Daily Score Rollover with Penalties

Problem: Implementing midnight rollover logic where remaining time converts to bonus points but overtime (screen time exceeding daily limit) applies penalties, with scores carrying forward to next day.

Solution: Created DailyScoreCarryoverService Kotlin service scheduled via AlarmManager with exact midnight trigger (accounting for timezone). Implemented complex calculation: first 40 minutes remaining = 67 points/min, 40-60 minutes = 40 points/min, 60+ minutes = 20 points/min (diminishing returns). Overtime penalty = 50 points/min, potentially creating negative scores that reduce next day's starting score.

Debugging Complexity: Rollover bugs were most difficult to debug due to 24-hour test cycles. Solution: Added comprehensive logging with [CARRYOVER-TRACE] tags showing complete calculation flow, before/after values, and verification steps. Created manual date-change testing procedures and test scenarios documentation.

Challenge 2: Persistent Foreground Service with Timer

Problem: Android aggressively kills background processes. Needed truly persistent timer that survives app closures, device sleep, and aggressive battery optimization.

Solution: Implemented foreground service (cannot be killed by system) with persistent notification showing real-time countdown. Used AlarmManager with exact alarms for critical timing events. Implemented service restart on device boot via BroadcastReceiver. Added "protection" layer detecting service death and auto-restart.

Edge Cases Handled: Device reboot (persist timer state, restore on boot), app force-stop (detect and handle gracefully), battery optimization (request exemption, inform user), Android 12+ exact alarm restrictions (request permissions), timezone changes (recompute midnight triggers).

Challenge 3: Negative Scores Not Displaying

Problem: When users had overtime penalties creating negative scores, UI wasn't updating to show negative values—confusing UX.

Solution: Traced through entire score persistence flow (Kotlin AsyncStorage write → JavaScript AsyncStorage read → Zustand store update → UI render). Added explicit logging at each step with "NEGATIVE ⚠️" indicators. Discovered AsyncStorage writes succeeding but reads returning stale data due to async timing. Fixed with forced notification update triggering immediate UI refresh after score calculation.

Learning: Cross-boundary debugging (native ↔ JavaScript) requires comprehensive logging at all boundaries. React Native's async nature demands careful state synchronization.

Challenge 4: Overtime Not Clearing for Large Values

Problem: Overtime would reset correctly for <1 hour but not for >2 hours, leaving notification showing incorrect overtime values.

Solution: Added [RESET-TRACE] logging showing before/after overtime values. Discovered overtime WAS resetting in service but notification wasn't updating. Root cause: notification update occurring before AsyncStorage write completed. Fixed by forcing notification update AFTER score calculation with verification of success. Added broadcast to UI components forcing immediate re-query of timer state.

Advanced Financial Analysis: Comparative Performance Study of Meme Stocks vs. Traditional Safe-Haven Assets

Executive Summary

This comprehensive financial analysis examines the risk-return profiles, volatility characteristics, and correlation patterns of meme stocks (AMC Entertainment Holdings, GameStop Corp., MicroStrategy Inc.) compared to traditional stable assets (Walmart Inc., Gold, USD) over a five-year period (2020-2025). The study employs multiple analytical frameworks to demonstrate how visualization methodology significantly impacts interpretation and decision-making in investment analysis.

Research Objectives & Methodology

This analysis was designed to address three primary research questions:

  • Risk-Return Analysis: How do speculative meme stocks perform relative to traditional stable assets across multiple performance metrics?
  • Volatility Assessment: What is the magnitude and pattern of price volatility in meme stocks compared to safe-haven assets, and what implications does this have for portfolio construction?
  • Correlation Dynamics: To what extent do meme stocks move independently of traditional market indicators (Gold, USD), and what does this reveal about their underlying price drivers?

Data Sources & Preparation

Data Collection: Historical price data was aggregated from multiple financial data providers including Yahoo Finance and Bloomberg Terminal, covering daily closing prices, trading volumes, and market capitalizations for the period January 2020 through January 2025. The dataset comprises approximately 7,500 individual data points across six instruments.

Data Cleaning & Validation: Implemented rigorous quality control procedures including:

  • Identification and handling of missing values through forward-fill methodology for non-trading days
  • Adjustment for stock splits and dividends to ensure price continuity
  • Validation of data integrity through cross-reference with multiple sources
  • Outlier detection using statistical methods (z-scores > 3 flagged for manual review)
  • Normalization of currency values to ensure comparable analysis across asset classes

Analytical Framework: The Importance of Visualization Design

A central theme of this analysis is demonstrating how visualization methodology profoundly impacts interpretation and decision quality. The initial visualizations presented deliberately employ simplified approaches that, while technically accurate, obscure critical information necessary for informed investment decisions. Subsequent visualizations demonstrate best practices in financial data presentation.

Critical Insight: Simplistic visualizations can lead to erroneous conclusions by omitting temporal context, volatility information, and risk-adjusted performance metrics. Professional financial analysis requires multi-dimensional visualization approaches that reveal the complete risk-return landscape.

Part 1: Limitations of Simplified Visualizations

The following two visualizations illustrate common pitfalls in financial data presentation and serve as cautionary examples of how incomplete analytical approaches can mislead investors.

Figure 1.1: Raw Price Comparison - Meme Stocks vs. Walmart

Visualization Description: This line graph displays absolute price movements for four equity instruments across a five-year observation period. Each line represents daily closing prices without normalization or adjustment for initial investment amounts.

Technical Analysis:

  • GME Short Squeeze Event (January 2021): GameStop experienced an unprecedented price surge from approximately $17.25 to a peak of $483, representing a 2,700% increase within days. This event, driven by coordinated retail investor activity on Reddit's r/WallStreetBets, forced institutional short-sellers to cover positions, creating a feedback loop that temporarily decoupled GME's price from fundamental valuations.
  • AMC Sympathy Rally: AMC Entertainment followed a similar but attenuated pattern, rising from sub-$5 levels to approximately $72 in June 2021. The correlation coefficient between AMC and GME during this period exceeded 0.85, indicating strong co-movement driven by shared sentiment rather than fundamental factors.
  • MSTR Bitcoin Correlation: MicroStrategy's price trajectory closely mirrors Bitcoin market cycles, with notable peaks during cryptocurrency bull markets. The company's strategy of using corporate treasury to purchase Bitcoin creates direct exposure, resulting in a correlation coefficient of approximately 0.78 with BTC/USD.
  • WMT Fundamental Stability: Walmart demonstrates characteristic blue-chip stability with a compound annual growth rate (CAGR) of approximately 6.2% over the period, driven by consistent revenue growth and operational efficiency improvements.

Critical Limitations of This Visualization:

  • Scale Distortion: The extreme volatility in meme stocks compresses Walmart's price movements to near-invisibility, making it impossible to assess WMT's performance trajectory or risk profile from this chart alone.
  • Lack of Risk Context: Absolute price movements do not convey downside risk, maximum drawdown periods, or volatility clustering—critical factors for risk management.
  • Missing Investment Perspective: Without normalization to initial investment amounts, investors cannot determine actual returns or compare cost-based performance across assets.
  • Temporal Context Deficiency: The visualization lacks annotations for significant market events, regulatory changes, or company-specific catalysts that drove price movements.

Professional Recommendations for Improvement:

  • Implement logarithmic scaling on the Y-axis to make percentage changes comparable across vastly different price ranges
  • Add event annotations with vertical lines marking key dates (short squeeze, earnings releases, regulatory announcements)
  • Include confidence bands or volatility envelopes (±1 standard deviation) to visualize risk ranges
  • Consider dual Y-axes or separate panels for assets with fundamentally different price scales
  • Supplement with a volatility index overlay (e.g., 30-day rolling standard deviation) to highlight risk periods

Figure 1.2: Average Price Comparison Across Asset Classes

Visualization Description: This bar chart presents arithmetic mean prices for six instruments over the entire observation period. Values displayed: Gold ($1,944.59), MSTR ($387.13), GME ($62.22), WMT ($54.92), AMC ($13.63), USD ($1.04).

Technical Analysis:

  • Gold's Premium Valuation: Gold's mean price of $1,944.59 per troy ounce reflects its role as a monetary metal and inflation hedge. The relatively high value correlates with periods of economic uncertainty, particularly during the COVID-19 pandemic and subsequent inflationary environment.
  • Meme Stock Averages Inflated by Spike Events: GME's average of $62.22 is heavily skewed by the January 2021 short squeeze. The median price would provide a more representative measure of central tendency, as the arithmetic mean is disproportionately influenced by extreme outliers. Statistical analysis shows GME's price distribution is highly right-skewed (skewness coefficient ≈ 4.8).
  • WMT's Consistent Mid-Range Valuation: Walmart's average price of $54.92 reflects stable, mature-company valuation with low deviation from mean. The coefficient of variation (standard deviation / mean) for WMT is approximately 0.18, compared to 2.4+ for meme stocks.
  • USD Stability as Base Currency: The USD index averaged $1.04, demonstrating the dollar's role as a stable reference point with minimal volatility relative to other instruments analyzed.

Critical Limitations of This Visualization:

  • Incomparable Asset Classes: Comparing absolute prices across equities, commodities, and currencies is analytically meaningless. A stock at $10 is not inherently "cheaper" or "better" than one at $100—what matters is percentage returns and risk-adjusted performance.
  • Mean Sensitivity to Outliers: Using arithmetic means for highly volatile assets produces misleading central tendency measures. Median or geometric mean would provide more robust estimates.
  • Absence of Volatility Information: The chart provides no indication of price dispersion, standard deviation, or range—critical factors that distinguish speculative from stable assets.
  • Temporal Aggregation Loss: Averaging across five years obscures important temporal patterns, trend changes, and regime shifts in market behavior.
  • No Risk-Return Framework: Investors care about returns relative to risk (Sharpe ratio, Sortino ratio), not absolute price levels. This visualization provides neither.

Professional Recommendations for Improvement:

  • Replace absolute prices with percentage returns normalized to a common starting point (e.g., $100 invested in each asset)
  • Include error bars representing ±1 standard deviation to visualize price dispersion and volatility
  • Add box plots showing median, quartiles, and outliers for each asset to reveal distribution characteristics
  • Implement separate panels for different asset classes or use normalized indices to enable meaningful comparison
  • Supplement with risk-adjusted metrics (Sharpe ratio) displayed alongside or as color coding

Urban Transportation Analytics: Comprehensive Analysis of TTC Traffic Collisions and Subway Delay Correlation

Executive Summary

This project investigates the relationship between surface-level traffic collisions and underground subway service disruptions within Toronto's extensive public transportation network. Using advanced SQL techniques for data preparation and Tableau for interactive visualization, this analysis examines over 250,000 collision records and 75,000 subway delay incidents across a multi-year period to test the hypothesis that surface traffic conditions meaningfully impact subway operational performance.

Research Context & Business Significance

The Toronto Transit Commission (TTC) serves approximately 1.8 million daily riders across North America's third-largest public transit system. Understanding the factors that drive service delays is critical for operational optimization, resource allocation, and infrastructure investment prioritization. This analysis addresses a fundamental question: Do surface-level traffic patterns and collision events impact underground subway reliability, or are delays primarily driven by internal operational factors?

The business implications are substantial. If surface traffic significantly influences subway performance, investments in traffic management and collision reduction could yield subway reliability improvements. Conversely, if delays are operationally driven, resources should focus on infrastructure maintenance, equipment upgrades, and process optimization.

Data Architecture & Preparation Methodology

Data Sources:

  • Traffic Collision Database: City of Toronto Open Data Portal, containing 267,543 reported collision records from 2016-2023 with fields including location coordinates, time stamps, collision severity, contributing factors, and environmental conditions.
  • TTC Subway Delay Data: TTC operational database documenting 78,921 delay incidents across all subway lines, including delay duration (minutes), cause classification, station location, time of occurrence, and impact on service.

Advanced SQL Data Preparation Pipeline:

  • Data Quality Assessment: Identified 12,847 collision records (4.8%) with missing or invalid timestamps, 3,204 records (1.2%) with location coordinate errors outside Toronto boundaries, and 891 subway delay records (1.1%) lacking cause classifications. Implemented systematic cleaning protocols to handle each category.
  • Temporal Standardization: Converted all timestamps to UTC timezone, created derived fields for hour-of-day (0-23), day-of-week (Monday-Sunday), month, and year. Applied SQL window functions to calculate rolling averages and identify temporal patterns.
  • Spatial Join Operations: Utilized PostGIS extensions to perform spatial joins between collision coordinates and TTC station proximity zones (500m radius). This enabled analysis of whether collisions near subway stations showed stronger correlation with delays.
  • Aggregation & Feature Engineering: Created hourly and monthly aggregation tables using GROUP BY operations, calculated collision density metrics per square kilometer, derived rush-hour classifications (7-10 AM, 4-7 PM), and engineered lagged variables to test whether collisions predict delays with temporal lags.
  • Statistical Metric Computation: Implemented SQL window functions to calculate 30-day rolling averages, standard deviations, and percentile rankings. Created correlation matrices using SQL statistical functions to quantify relationship strength between collision frequency and delay occurrences.

Data Validation Procedures: Cross-referenced collision data against Toronto Police Service reports (98.7% alignment), validated subway delay counts against TTC monthly operational reports (99.2% accuracy), and performed temporal consistency checks to ensure no duplicate entries or timestamp anomalies.

Analytical Methodology & Statistical Framework

Hypothesis Testing Approach:

  • Primary Hypothesis (H1): Surface traffic collision frequency positively correlates with subway delay occurrence and duration, with correlation coefficient ρ > 0.4 considered practically significant.
  • Null Hypothesis (H0): No meaningful correlation exists between traffic collisions and subway delays (ρ ≈ 0), suggesting independent operational systems.
  • Statistical Tests Applied: Pearson correlation analysis for linear relationships, Spearman rank correlation for non-linear associations, time-lagged cross-correlation to identify delayed effects, and regime-based subsetting (rush hour vs. off-peak) to detect context-dependent relationships.

Temporal Analysis Framework:

  • Hourly analysis (24-hour cycle) to identify circadian patterns and rush-hour dynamics
  • Weekly analysis (Monday-Sunday) to detect weekday/weekend behavioral differences
  • Monthly analysis to identify seasonal trends, weather effects, and annual patterns
  • Year-over-year comparison to control for long-term trends and infrastructure changes

Behavioral Analytics & Marketing Strategy: Cyclistic Bike-Share User Segmentation Study

Executive Summary

This capstone project for the Google Data Analytics Professional Certificate examines usage patterns across two distinct customer segments within Cyclistic, a Chicago-based bike-sharing service. Through comprehensive analysis of 5.7 million ride records, this study quantifies behavioral differences between casual riders and annual members, identifies conversion opportunities, and develops data-driven marketing strategies to increase membership subscriptions—the primary driver of long-term revenue stability and business growth.

Business Context & Strategic Importance

Cyclistic operates a fleet of 5,824 bicycles across 692 stations throughout Chicago, serving both casual users (single-ride and day-pass purchasers) and annual members (subscription holders). The company's financial analysis reveals that annual members are significantly more profitable than casual riders due to predictable recurring revenue, reduced transaction costs, higher lifetime value, and lower customer acquisition costs per ride.

The executive team has set a strategic objective: maximize annual memberships rather than focusing solely on casual ridership growth. However, converting casual riders to members requires deep understanding of behavioral differences, usage patterns, and value propositions that resonate with each segment. This analysis provides the empirical foundation for targeted marketing strategies designed to drive conversion.

Research Questions & Analytical Objectives

  • Behavioral Differentiation: How do casual riders and annual members differ in their usage patterns, ride characteristics, and temporal preferences?
  • Conversion Opportunity Identification: Which casual rider behaviors indicate latent demand for membership-style usage patterns?
  • Marketing Segmentation: What distinct casual rider sub-segments exist, and what messaging/incentives would most effectively drive conversion for each?
  • Seasonal Optimization: When do conversion campaigns have the highest likelihood of success based on usage pattern alignment?

Data Infrastructure & Analytical Methodology

Dataset Specifications:

  • 12 months of ride transaction data (January 2023 - December 2023)
  • 5,738,612 total ride records analyzed
  • Key variables: ride_id, rideable_type, start/end timestamps, station locations, geographic coordinates, member status
  • Data source: Cyclistic internal database (fictional scenario based on real Divvy bike-share data structure)

Comprehensive Data Preparation Process:

  • Data Import & Consolidation: Imported 12 separate monthly CSV files into Excel using Power Query, establishing unified schema. Addressed field name inconsistencies across months and standardized data types.
  • Data Quality Assessment: Identified 127,439 records (2.2%) with missing start/end station information, 8,732 records (0.15%) with implausible ride durations (>24 hours or <1 minute), and 3,208 duplicate ride_id entries. Implemented systematic exclusion criteria based on data reliability requirements.
  • Feature Engineering: Calculated ride_length_minutes = (end_time - start_time) in minutes, extracted day_of_week (Monday-Sunday), created month variable for seasonal analysis, engineered time_of_day categories (Early Morning, Morning Rush, Midday, Evening Rush, Night), and calculated straight-line distance between start/end coordinates using Haversine formula.
  • Outlier Treatment: Applied statistical outlier detection using IQR method. Rides with duration > Q3 + 1.5*IQR (>78 minutes) were flagged and analyzed separately as "extended usage" segment. Confirmed these were legitimate rides, not data errors, through manual sampling.
  • Validation Procedures: Cross-checked aggregate statistics against Cyclistic's published monthly reports (98.4% alignment), validated station coordinates against official station registry, and performed temporal consistency checks ensuring sequential timestamp logic.

Statistical Analysis Framework

Analysis employed multiple statistical techniques to ensure robust, reproducible findings:

  • Descriptive Statistics: Calculated mean, median, mode, standard deviation, and quartile distributions for ride length by user type
  • Comparative Analysis: Performed two-sample t-tests to confirm statistical significance of observed differences (all comparisons p < 0.001)
  • Temporal Pattern Recognition: Applied time-series decomposition to isolate trend, seasonal, and day-of-week effects
  • Segmentation Analysis: Used k-means clustering to identify behavioral sub-segments within casual rider population
  • Correlation Analysis: Examined relationships between ride length, frequency, and time-of-day preferences
