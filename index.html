BrainBites: Gamified Learning & Screen Time Management App

Android app React Native + TypeScript Kotlin native modules Firebase + AdMob
Executive summary: BrainBites combines device-wide screen time tracking with gamified quizzes to help students reduce phone overuse while reinforcing learning. Built from scratch with a production-minded architecture.
  • Frontend: React Native 0.80.2 + TypeScript, reusable screens, custom hooks, smooth UI animations
  • Native Android: Foreground services, UsageStatsManager, persistent notifications, AlarmManager scheduling
  • Architecture: service layer separating UI, business logic, persistence, and analytics
  • Data: AsyncStorage persistence, version-aware updates, consistency-focused state flow
  • Analytics: Firebase events supporting retention and funnel analysis
Key insight: the hardest problems were reliability and lifecycle edge cases: keeping timer state correct across app closures, device sleep, and day transitions.
Deep dive: technical highlights and challenges
Challenge: daily rollover with bonuses and penalties (including negative scores) and correct UI refresh across native and JS layers.
  • Foreground timer service: persistent countdown with notification display and protection against OS process kills
  • Rollover service: midnight scheduling with verified persistence writes and post-write UI refresh triggers
  • Cross-boundary debugging: structured logging at native-to-JS boundaries to track state updates end to end
Outcome: a production-ready app foundation with robust state and lifecycle behavior, designed to scale with features and analytics-driven iteration.

Urban Transportation Analytics: TTC Collisions vs Subway Delays

SQL Tableau Hypothesis testing Operational recommendations
Executive summary: tested whether surface collision patterns meaningfully correlate with subway delays. Result: evidence supports statistical independence, with delays driven primarily by internal operational factors.
Methods and conclusions
  • Feature engineering by hour, weekday, month, and season
  • Correlation and lag testing, plus multivariate regression framing
  • Cause code analysis to identify primary delay drivers
  • Operational focus recommendation: early morning and late evening reliability windows
